# Database seeders

Seeders use MikroORM `@mikro-orm/seeder` and Faker.js (`fakerRU`) to create deterministic initial/demo data.

## Run

```bash
pnpm nx run api:migration:up
pnpm nx run api:seed
```

`api:seed` is idempotent and can be re-run on an existing database.

## Fresh local reset

```bash
pnpm nx run api:seed:fresh
```

This target runs migrations down/up and then executes `DatabaseSeeder`.

## Current seed order

1. `IamSeeder` — default owner, managers, masters.
2. `CatalogSeeder` — service categories, services, body-type pricing.
3. `CrmSeeder` — demo client IAM accounts in batches of 500.

## Demo credentials

- Owner email: `owner@studio.local`
- Manager emails: `manager1@studio.local`, `manager2@studio.local`
- Master emails: `master1@studio.local` ... `master5@studio.local`

The seeded password hashes are placeholders. For real logins, replace them with hashes generated by the application's password hashing adapter.

## Batching rule

Large seeders must flush and clear MikroORM Identity Map after each batch:

```ts
await em.flush();
em.clear();
```

Use batch sizes between 500 and 1000 rows for large datasets.
